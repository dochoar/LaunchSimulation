# Example: CodeReview AI

This example simulates the market launch of **CodeReview AI** — an AI-powered code review assistant priced at $29/month — against a swarm of 50 synthetic developer personas.

It demonstrates the full LaunchSim workflow end-to-end.

---

## Files

| File | Purpose |
|------|---------|
| `product_brief.md` | The product definition: problem, solution, pricing, differentiators |
| `run_simulation.py` | Script that runs the full workflow via the API |
| `sample_output.json` | Pre-computed realistic output — read this to understand what the system produces |

---

## Run the demo script

### Option A — Preview sample output (no backend needed)

```bash
python examples/codereview-ai/run_simulation.py --sample
```

This loads `sample_output.json` and prints metrics, personas, social posts, and insights without touching the backend.

### Option B — Run a live simulation

Make sure the backend is running:

```bash
npm run dev        # starts both frontend (3000) and backend (8000)
```

Then:

```bash
python examples/codereview-ai/run_simulation.py
```

The script will:
1. Submit the CodeReview AI product to the simulation engine
2. Poll until the 50-agent simulation completes (~2-5 min depending on your LLM)
3. Print metrics, personas, objections, social posts, and insights
4. Interview the first agent who purchased the product

## What the simulation produces

### Personas
50 synthetic developer profiles with demographics, archetypes (early adopter, skeptic, price-sensitive, etc.), willingness-to-pay, cognitive biases, and a biographical narrative. Each persona independently decides whether to ignore, view, click, or purchase based on their profile vs. the product's price and pitch.

### Interactions
Each of the 50 agents generates one interaction event across a 72-hour window: `ignored`, `viewed`, `clicked`, `read`, `purchased`, `abandoned`, or `shared`. Purchasing agents have WTP ≥ price. Skeptics and price-sensitive personas are far less likely to convert.

### Social posts
Active agents post to Twitter, Reddit, or Product Hunt. Posts reflect their persona — junior devs ask for discounts, security engineers flag privacy concerns, early adopters evangelize. Sentiment scores range from -1 to +1.

### Metrics
- Conversion funnel (views → clicks → purchases)
- Average sentiment across all posts
- Top 5 objections ranked by frequency
- 72-hour adoption curve (hourly cumulative data points)
- 4-5 strategic insights generated by the Chronicler agent

### Interviews (live only)
After simulation, you can ask any generated persona any question in natural language. The Interviewer agent retrieves their ChromaDB memory via RAG and responds in first person, in character.

## Why CodeReview AI?

It was chosen as a realistic example because:

- **Clear pain point** — PR review bottlenecks are universally recognized in dev teams
- **Believable price** — $29/mo is a real SaaS price point with genuine friction
- **Mixed market** — produces varied adoption: enthusiasts buy immediately, price-sensitive devs bounce, security skeptics ignore it entirely
- **Rich social commentary** — developers are vocal on Twitter and Reddit, generating realistic social posts with real objections
- **Actionable insights** — the output surfaces concrete, testable hypotheses (add a cheaper tier, get SOC 2, add referral program)
